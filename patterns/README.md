# 设计模式

## 创建型

### 单例模式

定义：一个类只允许创建一个对象(实例)，那这个类就是单例类，创建单例类的模式就是单例模式。

用处：从业务概念上，有些数据在系统中只应该保存一份。

难点： 如何实现唯一性

1. 进程之间唯一
2. 线程之间唯一
3. 集群之间唯一

问题

1. 对OOP的特性支持不友好
2. 会隐藏类之间的依赖关系

- 降低可读性
- 通过构造函数、参数传递等方式声明的类之间的依赖关系，

4. 对代码扩展性不友好
5. 如果一天需要创建多个实例会比较麻烦
6. 对代码的可测性不友好
7. 不支持有参数的构造函数

## 设计原则

- KISS
  - 尽量保持简单
    - 逻辑复杂度
    - 实现难度
    - 代码可读性
  - 常见原则
    - 不要使用同事可能不懂得技术来实现代码
    - 不要重复造轮子，善于使用已经有的工具
    - 不要过度优化 YAGNI
- DRY
  - 不要重复自己
  -
- LOD
- SOLID
  - 单一职责
  - 开闭原则
  - 里式替换
  - 接口隔离
  - 依赖倒置
